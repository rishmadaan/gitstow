**Date:** 2026-07-12
**Method:** Live browser session (Chrome, real repos seeded in every state: clean, modified,
staged, untracked-only, behind, diverged, frozen, missing, local-only/no-upstream, across

## Findings

| ID | Severity | Finding | Where |
|----|----------|---------|-------|
| U1 | Critical | **Search box, workspace filter, and "Hide frozen" are decorative** — rendered with promising tooltips, wired to nothing (no ids, handlers, or hx-attrs). v0.2.0 mockup leftovers. | `dashboard.html:32-37` |
| U2 | Critical | **"Save preferences" fires `alert('Settings save lands later')`** — no POST route exists; the SSH checkbox/host select silently discard input. Also missing: `parallel_limit`, `clone_timeout` fields (page claims to mirror `config show`). | `settings.html:14`, `pages.py:35` |
| U3 | Important | **Dashboard dead without internet:** htmx from unpkg CDN (no SRI), fonts from Google CDN. Every button on a localhost tool depends on the network. | `base.html:8`, `app.css:4` |
| U4 | Important | **Local-only repos (no upstream) fail every Pull all** with raw git stderr; Remote Δ shows the same "—" as in-sync; Pull tooltip claims "already up to date". Model knows `no-upstream`; bulk workers don't consult it. | `web/routes/repos.py:_pull_if_safe`, `cli/pull.py:_pull_one_repo`, `dashboard.py:_delta` |
| U5 | Important | **Native `confirm()`/`alert()` on six actions** (Pull all, Shutdown, row Remove, drawer Remove/Delete, workspace Remove) — jarring, and hard-freezes browser automation of an AI-first tool. | `base.html:46`, `dashboard.html:58`, `partials/repo_row.html:72`, `_repo_drawer.html:94,97`, `workspaces.html:51` |
| U6 | Important | **Web collection import ignores recorded workspaces** (CLI honors them since v0.3.0) — duplicate parser + no per-entry routing. Cross-surface drift. | `web/routes/collection.py:82,148-236` |
| U7 | Important | **Pull all spinner never stops** — persists minutes after "complete", across interactions. Root cause TBD (`.htmx-request` seemingly never cleared). | `dashboard.html:52-58`, `app.css:904-906` |
| U8 | Moderate | **Zero responsive rules** (no `@media`, no `overflow-x` in 922 CSS lines) — breaks below ~1100px; half-monitor is a core use case. Decision: target 720px floor. | `app.css` |
| U9 | Moderate | **"Clone & catalog" gives no pending feedback** — synchronous POST hangs the page for the whole clone. | `add_repo.html` |
| U10 | Moderate | **A11y gaps:** "Hide frozen" checkbox announces as "on"; disabled Pull buttons aren't `disabled`; no `:focus-visible` treatment on menu/summary. (Row menu itself is a real `<details>/<summary>` — keyboard OK.) | `dashboard.html:37`, `partials/repo_row.html` |
| U11 | Moderate | **Honesty/polish batch:** raw ISO+microseconds timestamp on detail page; `last_fetched` surfaced nowhere; diverged+missing lumped into the "conflict" chip (decision: split into 3 chips); "5 processed" vs chips summing 7; help-dialog statuses in a cut-off nested scrollbox; unstyled native file input; static green LIVE dot; workspace path boxes styled like editable inputs. | various |

## Decisions (locked 2026-07-12)

1. Hero metrics split into three chips: `conflict` (dirty+behind), `diverged`, `missing` — each rendered only when nonzero.
2. Responsive floor is **720px** (half-monitor). Phone layouts out of scope.
3. Execution may use **Codex Mode** for mechanical, spec-complete tasks; taste-heavy UI/copy tasks stay with Claude subagents (per the Codex-mode doctrine).
